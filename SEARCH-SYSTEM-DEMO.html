🔄 Data Flow

            

1. User types in topbar search → JavaScript input event listener
2. 120ms debounce delay to avoid excessive requests
3. JS calls POST /search/suggest?q=...&limit=10
4. Backend CoreTopbarBridge::searchResults() scores results
5. JSON response: {q, count, items: [{label, type, url, answer, ...}]}
6. JS renders suggestions with:
   - Type badge (page/settings/module)
   - Title + description
   - Answer snippet (highlighted relevance)
   - Input parameters (expandable actions)
   - Direct URL link
7. User navigates with Arrow keys or clicks → Window redirects
8. Full results page available at /search?q=...

⚖️ Query Scoring Algorithm

            

                
                    

                        Match Type
                        Points
                        Description
                    

                
                
                    

                        Label starts with query
                        +48
                        Best match - user searching for exact page name
                    

                    

                        Label contains query
                        +30
                        Good match - query is in title
                    

                    

                        Keywords contain query
                        +20
                        Relevant - query matches meta keywords
                    

                    

                        Description contains query
                        +16
                        Related - query in page description
                    

                    

                        Input parameters match
                        +14
                        Action-oriented - query matches form inputs
                    

                    

                        Answer contains query
                        +12
                        Contextual help - query in answer text
                    

                    

                        URL contains query
                        +8
                        Weak match - query in URL path
                    

                
            

            
Tie-breaking: When scores are equal, results are sorted alphabetically by label

        


        
        

            
🔗 Integration Points

            
            
Frontend Integration (HTML)

            
Topbar search form renders with these data attributes:

            
<form
    data-cat-search-form
    data-cat-search-items="[{...}]"
    data-cat-search-endpoint="/admin/search/suggest"
    data-cat-search-results-url="/admin/search"
    action="/admin/search"
>
    <input data-cat-search-input name="q" type="search">
    <button type="submit">Rechercher</button>
    <div data-cat-search-suggest hidden></div>
</form>


            
JSInitialization

            
The catmin-topbar.js runs when page loads:

            
document.addEventListener('DOMContentLoaded', () => {
    initTopbarSearch(); // Scans for [data-cat-search-form]
});

// Or at end of file:
initTopbarSearch();


            
Backend Routes

            
Two new routes registered in admin/routes.php:

            
GET /admin/search/suggest?q=...&limit=10
    → CoreTopbarBridge::searchResults()
    → Returns JSON with scored results

GET /admin/search?q=...
    → View: admin/views/search/index.php
    → Shows full-page Google-style results
